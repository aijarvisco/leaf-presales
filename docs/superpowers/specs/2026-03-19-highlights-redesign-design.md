# Highlights Section Redesign — Design Spec

**Date:** 2026-03-19
**Status:** Approved

---

## Overview

Redesign the Highlights section of the Nissan Leaf presales landing page to be a premium, high-impact showcase. The section must contrast sharply with the dark Hero section, feel tech-forward and electric, and use an Apple-style center-focus carousel for card browsing.

---

## Goals

- Strong visual contrast with the Hero (white vs dark)
- Impactful, premium typography that feels tech/electric
- Generous whitespace to give the section authority and breathing room
- Apple-style center-focus carousel — active card full-size, adjacent cards peek in at reduced scale and opacity
- Bigger, more dominant cards with real Nissan Leaf imagery

---

## Typography — Global Change

### Font

Replace the current heading font (Geist Sans) with **Space Grotesk** across all page headings.

### Integration in `layout.tsx`

Load Space Grotesk using the variable pattern (same as Cormorant Garamond):

```ts
const SpaceGrotesk = Space_Grotesk({
  subsets: ['latin'],
  weight: ['400', '500', '600', '700'],
  variable: '--font-space-grotesk',
})
```

Apply the variable class to the `<html>` element alongside the existing Cormorant class:

```tsx
<html lang="pt" className={`${SpaceGrotesk.variable} ${cormorant.variable}`}>
```

The `<body>` keeps `GeistSans.className` for body text — no change there.

### Updates in `globals.css`

1. Add to `@theme`:
```css
--font-family-heading: var(--font-space-grotesk), system-ui, sans-serif;
```

2. Update the `h1, h2, h3, h4` base rule:
```css
h1, h2, h3, h4 {
  @apply font-heading font-bold tracking-tight;
}
```
Note: changes both the font family (`font-heading`) **and** the weight (`font-bold` = 700, replacing `font-semibold` = 600).

The Tailwind utility `font-heading` is generated by Tailwind v4 from the `--font-family-heading` token (following the same convention as `--font-family-sans` → `font-sans`). The name `font-heading` avoids any conflict with the native CSS `font-display` property.

This is a **global change** — all section titles across the page (Hero, Highlights, CTAs, etc.) inherit Space Grotesk automatically.

## Carousel — Layout

The carousel track is a single `motion.div` rendered as a flex row (`flex flex-row gap-6`). It is positioned inside a full-width overflow-hidden container. The track is translated on the x-axis to keep the active card centered in the viewport.

### Centering formula

Measure `cardWidth` at runtime using a `useRef` on the first card element and a `ResizeObserver` that updates the value on resize. Do not hardcode — let the DOM report the actual rendered width so both the `75vw` (mobile) and `60vw` (desktop) breakpoints work correctly.

```ts
const cardRef = useRef<HTMLDivElement>(null)
const [cardWidth, setCardWidth] = useState(0)

useEffect(() => {
  if (!cardRef.current) return
  const ro = new ResizeObserver(() => {
    setCardWidth(cardRef.current!.getBoundingClientRect().width)
  })
  ro.observe(cardRef.current)
  return () => ro.disconnect()
}, [])
```

The x offset to center card at `activeIndex`:

```ts
const gap = 24 // px (gap-6)
const viewportWidth = window.innerWidth
const targetOffset = (viewportWidth / 2) - (cardWidth / 2) - activeIndex * (cardWidth + gap)
```

On mount and on `activeIndex` change, animate the track `x` to `targetOffset` using the spring parameters below.

### Drag Constraints

Pass **computed pixel values** (not a container ref) to `dragConstraints`:

```ts
const maxRight = (viewportWidth / 2) - (cardWidth / 2)                                          // activeIndex = 0
const maxLeft  = (viewportWidth / 2) - (cardWidth / 2) - (cards.length - 1) * (cardWidth + gap) // activeIndex = last

<motion.div dragConstraints={{ left: maxLeft, right: maxRight }} dragElastic={0.1} ...>
```

Recompute these on `cardWidth` or viewport resize.

### Container

```tsx
<div className="w-full overflow-hidden">
  <motion.div
    className="flex flex-row gap-6"
    drag="x"
    dragConstraints={{ left: maxLeft, right: maxRight }}
    dragElastic={0.1}
    animate={{ x: targetOffset }}
    transition={spring}
    onDragEnd={handleDragEnd}
  >
    {cards}
  </motion.div>
</div>
```

The first card is NOT inset — the offset formula naturally positions the first card centered when `activeIndex = 0`, with the right card peeking in. On the last card, the left card peeks in.

## Carousel — Behavior

### Active Index Tracking

Use a discrete `activeIndex` state (not continuous). The active card changes when:
- The user releases the drag and the drag offset has crossed **50% of the card width** in either direction, **or**
- The drag velocity at release exceeds **500px/s** in either direction (flick gesture)

Clamp `activeIndex` between `0` and `cards.length - 1` — no looping.

### Drag Bounds & Edge Behavior

Drag is bounded using Framer Motion's `dragConstraints`:
- Left bound: offset when `activeIndex = cards.length - 1`
- Right bound: offset when `activeIndex = 0`

Set `dragElastic={0.1}` for a subtle spring-back at the edges (not a hard stop, not a full bounce).

### Spring Parameters (track animation)

```ts
{ type: 'spring', stiffness: 300, damping: 30, mass: 0.5 }
```

### Scale & Opacity per Card Position

Applied as discrete CSS transforms based on distance from `activeIndex`. Cards update their scale/opacity as `activeIndex` changes.

| Distance from active | Scale | Opacity |
|---|---|---|
| 0 (active) | `1.0` | `1.0` |
| ±1 (adjacent) | `0.92` | `0.60` |
| ±2 or more | `0.88` | `0.40` |

Transitions between states use:
```ts
{ type: 'tween', duration: 0.3, ease: 'easeInOut' }
```

### Inactive Card Overlay

Cards that are **not** the active card (`distance > 0`) get a `bg-white/20` overlay div positioned `inset-0` inside the card. This overlay fades in/out with the same `0.3s easeInOut` transition as the scale/opacity. There is no continuous gradient — it is a binary active/inactive overlay.

## Card Design

### Dimensions

| Property | Value |
|---|---|
| Width | `w-[75vw] md:w-[60vw]` |
| Aspect ratio | `aspect-[4/5]` |
| Border radius | `rounded-2xl` |
| Shadow | `shadow-xl` |
| Background | White |

### Layout

- Top **65%**: full-bleed image (`object-cover`)
- Bottom **35%**: content area with `p-8` padding

### Content Area

| Element | Style |
|---|---|
| Category label | All-caps, Space Grotesk, `font-medium` (500), `text-xs`, `tracking-widest`, muted gray (`text-gray-400`) |
| Feature title | Space Grotesk, `font-bold` (700), `text-2xl md:text-3xl`, near-black (`text-gray-900`) |
| Description | Geist Sans, `font-normal` (400), `text-base`, secondary gray (`text-gray-500`), `leading-relaxed` |

### `HighlightCard` Props Interface

```ts
interface HighlightCardProps {
  imageSrc: string
  imageAlt: string
  category: string   // all-caps label
  label: string      // feature title
  description: string
  isActive: boolean  // controls scale, opacity, and overlay
}
```

Note: `videoSrc` is **intentionally removed** in this rewrite. The carousel is image-only. If video support is needed in the future it should be re-introduced as a separate design decision.
